# Button（按钮）组件说明

## 一、组件概述

按钮组件用于触发具体行为或动作，通过点击执行操作，是交互流程中引导用户操作的核心基础组件，可根据场景需求，搭配不同样式、状态，适配多样交互场景。

## 二、构成元素

容器元素包括边框、背景、文案、图标等元素。

### 按钮类型（六种）

**填充按钮**

- 主色填充 + 白色文案
- 灰色填充 + 深色文案

**图标按钮**

- 填充按钮 + 左侧/右侧图标
- 仅图标（方形）

**描边按钮**

- 白底 + 灰色描边 + 文案
- 白底 + 主色描边 + 主色文案 + 图标
- 仅图标（方形）

**虚线按钮**

- 白底 + 灰色虚线描边 + 文案
- 白底 + 主色虚线描边 + 仅图标

**文字按钮**

- 纯文案（深色/主色）
- 文案 + 图标（深色/主色）

**组合按钮**

- 填充按钮 + 右侧下拉箭头（下拉操作）

## 三、交互状态

默认（Default）、悬停（Hover）、聚焦（Focused）、激活（Active）、禁用（Disabled）等状态，通过背景色、边框色、透明度等视觉变化提供反馈，强化交互逻辑。

## 四、尺寸适配

提供 Large（大）、Regular（常规）、Small（小）、Mini（迷你）等尺寸。

## 五、设计与应用原则

- **功能清晰**：文案、图标精准匹配操作，避免歧义，确保用户快速理解交互目的。
- **层级分明**：通过类型、尺寸、状态区分操作优先级（如填充按钮承载主操作，文字按钮承载次要操作），引导用户操作流程。
- **视觉统一**：同一产品内，按钮样式、交互反馈保持一致，降低用户学习成本，强化品牌体验。
- **场景适配**：结合业务场景（如表单提交、列表操作、弹窗交互），选择匹配的按钮类型、尺寸，平衡视觉表现与交互效率，提升界面可用性与美观度。
